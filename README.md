# Copyright Evidence Organizer

一个面向互联网平台版权维权场景的轻量级前端 Demo，用于整理版权案件材料、生成证据与归档清单，并辅助进行版权授权文件的初步核查。

本项目仅用于材料整理和审核线索提示，不构成法律意见；最终结论需由法务人员结合原件、业务背景和适用规则复核。

## 功能概览

### 版权案件材料整理

- 登记作品名称、权利人、侵权链接、平台、发现时间、截图情况、权属文件情况和处理进度
- 上传 `.docx`、`.txt`、`.csv` 线索文件并尝试提取案件信息
- PDF 文件暂时只记录文件名，不读取正文
- 自动生成案件登记表、证据材料清单、版权文件归档清单、案件进度统计表和待补材料提醒
- 支持导出案件 CSV

### 版权授权文件审核

- 支持授权书、版权登记证书、合作协议、营业执照、身份证明、链路授权文件等类型
- 根据文件类型生成审核 checklist
- 上传 `.docx`、`.txt`、`.csv` 文件并尝试提取授权相关字段
- 提取字段包括权利人/授权人、被授权人、作品名称、授权期限、授权范围、信息网络传播权、维权/投诉/诉讼授权、转授权和附件清单
- 对无法稳定识别的字段标记为“待人工确认”
- 自动生成授权文件审核台账和文件补正提醒
- 支持导出授权审核 CSV

## 使用方法

1. 打开 `index.html`。
2. 在顶部切换“版权案件材料整理”和“版权授权文件审核”两个模块。
3. 可以点击“载入样例”查看示例数据。
4. 上传 `.docx`、`.txt` 或 `.csv` 文件后，页面会在本地尝试解析正文并提取字段。
5. 核对自动提取结果后，可保存案件或审核记录。
6. 如需交接或归档，可导出 CSV。

## 本地运行

这是一个纯前端静态页面项目，不需要后端服务。

直接用浏览器打开：

```text
index.html
```

## 技术说明

- 使用 HTML、CSS 和原生 JavaScript 构建
- 使用浏览器本地能力读取 `.txt`、`.csv` 文件
- 对 `.docx` 文件，尝试在浏览器本地解压并读取 `word/document.xml`
- 数据默认保存在浏览器 `localStorage`
- 所有解析都在本地完成，不上传文件到服务器

## 注意事项

- 文件解析结果仅作为材料整理和审核线索
- 未识别字段会标记为“待人工确认”，不会直接判断为缺失
- PDF 正文解析暂未实现
- 最终审核结论需由法务人员复核
